## Пины

**Главный вывод по torch: Blackwell СОХРАНЯЕМ, но индекс должен быть `cu128`, а не `cu126`.**

Разбор (это правит предпосылку плана/спеки):

- План предполагал ставить torch с индекса **cu126** и, если свежий torch не заведётся, откатиться на `torch==2.6.0`. Проверка показала другое: сборка **cu126** вообще не содержит ядер для sm_120 — ни в свежем `2.13.0+cu126`, ни в `2.6.0+cu126`. `torch.cuda.get_arch_list()` для cu126 = `sm_50…sm_90`; на матумуле GPU падает `CUDA error: no kernel image is available for execution on the device`. То есть **откат на 2.6.0 Blackwell бы не спас — cu126 это тупик для RTX 50xx в принципе.**
- Сборка **cu128** решает вопрос: `torch==2.11.0+cu128` даёт `get_arch_list()` = `sm_75, sm_80, sm_86, sm_90, sm_100, sm_120`; матумул на карте проходит. `chatterbox-tts==0.1.7`, поставленный `--no-deps`, на этом torch **импортируется и синтезирует** без ошибок. gradio не ставился и не нужен.
- Вердикт: **свежий torch завёлся, torch 2.6.0 НЕ понадобился, Blackwell НЕ теряем.** Строку про «отсутствие RTX 50xx» в README (Задача 9) писать не нужно. Наоборот: поддержка Blackwell держится именно на индексе cu128 — **при откате на cu126 карта отвалится.**

Замечания для Задач 3–4 (важно, чтобы не наступить на грабли спайка):

- **numpy 2.x работает.** chatterbox-tts в метаданных требует `numpy<2.0.0`, но синтез на `numpy==2.4.6` прошёл без ошибок. Понижать не нужно.
- **torchaudio.save в 2.11 больше не пишет WAV без `torchcodec`** (кидает `ImportError: TorchCodec is required`). Это НЕ проблема продакшена: worker пишет WAV модулем `wave` (Задача 4) — так и оставить, `torchaudio.save` не использовать. В спайке для сохранения взят `soundfile==0.14.0` (приезжает как зависимость librosa).
- Конфликты пинов при установке зависимостей — ожидаемые и безвредные (chatterbox хочет `gradio==6.8.0`, `diffusers==0.29.0`, `transformers==5.2.0`, `safetensors==0.5.3`, `torch==2.6.0`); мы их сознательно перекрываем, движок с новыми версиями работает.
- Безвредные предупреждения в логах: `pkg_resources is deprecated` (perth), `russian_text_stresser not available - Russian stress labeling skipped` (опциональная простановка ударений, на синтез не влияет), `You are sending unauthenticated requests to the HF Hub`.

## Метрики

Замерено на RTX 5070 Ti, torch 2.11.0+cu128, фраза: «Ну и куда ты поехал, гений тактики? Вся команда смотрит на этот позор.»

- **sr (частота синтеза):** 24000 Гц.
- **VRAM:** после загрузки модели 3074 МБ; во время синтеза ~3124 МБ; **пик 3261 МБ (~3.2 ГБ)**. Это примерно вдвое меньше S1-mini (~6 ГБ) — цель спеки выполнена.
- **Загрузка модели:** v2 — 11.6 с, v3 — 14.5 с (разовая, при старте worker).
- **Время на фразу (по стилям):** 3.7–5.8 с. С запасом укладывается в `tts_max_age_s = 20`.

Разбивка по стилям (v2 / v3, секунды): neutral 5.5/5.8, angry 4.8/4.7, shouting 4.9/4.0, excited 4.5/4.3, laughing 4.0/4.4, confident 4.6/4.2, sad 3.7/4.2, whispering 4.5/3.9.

## Стили

**Вердикт пользователя (2026-07-12): 8 градаций звучат слишком похоже — пресеты сокращены до четырёх контрастных.** `EMOTION_MARKERS` в промпте LLM теперь `angry / excited / sad / whispering`; старые маркеры парсер вырезает и приводит к ближайшему (`_LEGACY` в markers.py) — ради персон, сохранённых в БД со старым списком.

Итоговая таблица (значения разведены по углам пространства):

| Маркер | exaggeration | cfg_weight |
|---|---|---|
| без маркера / neutral | 0.5 | 0.5 |
| angry (+shouting, disdainful) | 0.9 | 0.25 |
| excited (+laughing, chuckling, confident, surprised) | 0.75 | 0.35 |
| sad (+sighing) | 0.35 | 0.65 |
| whispering | 0.25 | 0.7 |

Контрольные wav новых значений — `out2/style_*.wav` (генерируются повторным прогоном спайка).
