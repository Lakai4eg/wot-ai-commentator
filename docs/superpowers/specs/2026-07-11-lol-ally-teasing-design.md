# LoL: разнообразие реплик и подколки союзников — дизайн

Дата: 2026-07-11. Статус: утверждён.

## Цель

Больше уникальных реплик на события League of Legends и новые поводы для
подколок — как самого стримера, так и его союзников. Сейчас союзники в
комментариях отсутствуют (маппер их игнорирует), LLM-реплики повторяют
любимые формулировки, а фолбэк-шаблоны содержат всего 2–4 варианта на событие.

## Обзор решения

Четыре независимых блока:

1. Маппер отслеживает счёт союзников и порождает новые события
   (`ally_feeding`, `ally_carrying`, `team_gap`) с антиспам-ограничениями.
2. Память получает таблицу союзников через тихий стимул `team_state`
   и добавляет заметных союзников в контекст каждой реплики.
3. Промпт: ротация «углов шутки» (аналог `_ADDRESS_STYLES`) и история
   последних озвученных реплик с запретом повторов.
4. Фолбэк-шаблоны: расширение до 8–12 вариантов на событие, шаблоны для
   новых событий, выбор без повтора последних использованных.

## 1. Новые события в маппере (`games/lol/mapper.py`)

Маппер читает счёт тиммейтов из снапшота `allPlayers` (поле `scores`:
kills/deaths каждого игрока команды стримера, кроме него самого). Имена
нормализуются существующим `_name_key`; чемпион — `championName`.

### `ally_feeding`

Союзник набрал 5 смертей; повторно — на каждых +3 (5, 8, 11…).
Payload: `{champion, deaths}`. Priority NORMAL, ttl 20 с.
Пороги отслеживаются на союзника: словарь «последний озвученный порог».

### `ally_carrying`

Два триггера, оба NORMAL, ttl 15 с:

- Журнальный `Multikill`, где `KillerName` — союзник (сейчас событие молча
  теряется). Payload: `{champion, label, count}` — те же лейблы
  `_MULTIKILL_LABELS`.
- Союзник опережает стримера на ≥5 киллов при ≥8 киллах у союзника.
  Срабатывает один раз за игру на союзника. Payload:
  `{champion, kills, my_kills}`. Мишень шутки — стример
  («кто-то играет вместо тебя»), не союзник.

### `team_gap`

Редкие «макро»-подколки, каждая не чаще одного раза за игру:

- К 10-й минуте (`gameTime ≥ 600`) стример 0/0/0, при этом у команды в сумме
  ≥5 киллов. Payload: `{kind: "spectator", team_kills}`.
- Команда отстаёт от противника на ≥10 суммарных киллов.
  Payload: `{kind: "behind", diff}`.

### Общие ограничения

- Глобальный интервал: не чаще одного события про союзников
  (`ally_feeding`/`ally_carrying`/`team_gap`) в 60 секунд — таймштамп
  последнего в маппере.
- Все трекеры сбрасываются в `_reset_game_trackers()`.
- Отсутствующие поля `scores` читаются как 0; игроки без команды/имени
  пропускаются.
- События противников не порождаются.

## 2. Контекст команды в памяти (`games/lol/memory.py`)

Маппер шлёт **тихий** стимул `team_state` (payload `{allies: [{champion,
kills, deaths}, ...], "silent": true}`) при изменении счёта любого союзника —
по образцу существующих silent-событий: обновляет память, не озвучивается.

`LolBattleMemory` хранит последнюю таблицу союзников. `lines()` добавляет
не более 2 строк и только про «заметных» союзников:

- фидер: ≥4 смертей → `союзник Yasuo: 2/7 — фидит`;
- керри: ≥6 киллов → `союзник Lee Sin: 9/1 — тащит`.

Так LLM может вплетать подколки союзников даже в реплики на события стримера.

## 3. Разнообразие LLM-реплик

### Углы шутки (`commentary/prompts.py` + модуль игры)

`GameModule` получает необязательное поле
`joke_angles: Callable[[], tuple[str, ...]] | None = None`.
`build_prompt` при наличии добавляет строку
`Угол шутки на этот раз: {random.choice(...)}.` рядом с обращением.

LoL-модуль отдаёт ~10 углов, например: подколоть союзника по ситуации;
самоирония ИИ без рук; обыграть текущий счёт; бытовая метафора; пафос
киберспортивного кастера; «чат уже печатает»; обыграть чемпиона противника;
похвала с подвохом; сухая констатация; обыграть ферму/варды.
WoT не меняется (поле опциональное).

### История реплик (`director.py` + `commentary/prompts.py`)

Директор хранит `deque(maxlen=8)` последних озвученных реплик (LLM и
фолбэк, обе игры — голос один). `build_prompt` получает параметр
`recent_lines` и добавляет блок:
`Твои последние реплики (не повторяй их формулировки, образы и шутки):`
со списком. Реплика попадает в историю после успешной постановки в озвучку.

## 4. Фолбэк-шаблоны и описания (`games/lol/flavor.py`)

- Каждый существующий список `_TEMPLATES` расширяется до 8–12 вариантов.
- Новые списки: `ally_feeding`, `ally_carrying` (отдельные формулировки для
  мультикилла союзника и для отрыва по киллам — ключ выбирается по payload),
  `team_gap` (по `kind`).
- `_EVENT_DESCRIPTIONS` — записи для новых событий с указанием мишени, напр.:
  `ally_feeding`: «Союзник стримера {champion} набрал уже {deaths} смертей —
  подколи его как кормильца команды; стримера не вини»;
  `ally_carrying`: «…мишень — стример, союзника хвали»;
  `team_gap`: описание по `kind`.
- Выбор фолбэка без повтора: помним последние 3 выданных варианта на тип
  события, выбираем из оставшихся (если вариантов меньше 4 — просто не
  повторяем последний). Состояние — модульное, на процесс.
- Блок `_FLAVOR` дополняется строкой про новые поводы: джунглер, фидящий
  союзник, керри — легитимные мишени дружеских подколок.

## Обработка ошибок

- Ошибки чтения `scores`/`allPlayers` не роняют обработку снапшота
  (существующий паттерн: try/except вокруг диспатча, дефолты 0).
- `team_state` с пустым списком союзников не порождается.
- Неизвестный `kind`/payload в фолбэках — деградация до общего списка события
  или «Без комментариев» (существующее поведение).

## Тесты

- `test_lol_mapper`: пороги `ally_feeding` (5, 8, 11), `ally_carrying` по
  мультикиллу союзника и по отрыву (один раз за игру), `team_gap` (по разу),
  глобальный 60-секундный интервал, сброс трекеров на новом матче,
  отсутствие событий для противников, тихий `team_state` при смене счёта.
- `test_lol_memory`: строки заметных союзников (фидер/керри), лимит 2 строки,
  сброс на `battle_start`.
- `test_lol_flavor`: описания новых событий, шаблоны новых событий,
  выбор без повтора последних 3.
- `test_prompts` (или новый): блок истории реплик, строка угла шутки,
  отсутствие блока при пустой истории/None joke_angles (WoT-совместимость).
- `test_director`: реплика попадает в историю после озвучки.

## Вне рамок

- Роли союзников (`position`) — поле часто пустое вне ранкеда; не используем.
- Подколки конкретных вражеских игроков по нику.
- Изменения WoT-модуля (история реплик затрагивает его автоматически —
  это улучшение общего ядра, поведение WoT иначе не меняется).
