# Замена озвучки: OpenAudio S1-mini → Chatterbox Multilingual

Дата: 2026-07-12
Статус: согласовано, готово к плану

## Зачем

Текущая озвучка (OpenAudio S1-mini через fish-speech) звучит нормально, но
негодна по двум причинам:

1. **Лицензия.** OpenAudio S1 распространяется под CC-BY-NC — некоммерческая.
   Для монетизируемого стрима и распространения в сборке это неприемлемо.
2. **Хрупкость установки.** fish-speech тянет gradio, wandb, protobuf; пины
   разъезжаются под каждым новым релизом transformers; для импорта нужен
   костыль — пустой `.project-root` в корне рантайма.

Дополнительное требование: модель не должна быть тяжёлой — она делит видеокарту
с игрой.

## Выбор модели

Три требования сразу — русский, управляемая эмоция, клон голоса с записи —
закрывают только две модели.

| Модель | Лицензия | Размер / VRAM | RU | Эмоция | Клон |
|---|---|---|---|---|---|
| **Chatterbox Multilingual** | **MIT** | **0.5B**, ~2–3 ГБ | да | `exaggeration` + референс | да, zero-shot |
| S1-mini (текущая) | CC-BY-NC ✗ | 3.6 ГБ, ~6 ГБ VRAM | да | маркеры | да |
| Qwen3-TTS Base (0.6B/1.7B) | Apache | 0.6–1.7B | да | **нет** ✗ | да |
| Qwen3-TTS CustomVoice / VoiceDesign | Apache | 1.7B | да | `instruct` | **нет** ✗ |
| XTTS-v2 | non-commercial ✗ | — | да | почти нет | да |
| IndexTTS-2, CosyVoice2 | — | — | **нет** ✗ | да | да |

**Решение: Chatterbox Multilingual** (Resemble AI, MIT). Русский входит в 23
языка, клон zero-shot по одному wav (**транскрипт не нужен**), выразительность —
числовой ручкой `exaggeration`, вдвое легче нынешней модели по VRAM.

Ограничения, принятые сознательно:

- `exaggeration` — интенсивность подачи, а не выбор эмоции. Категориальные
  эмоции («злой», «шёпот») достигаются подбором референса — см. «Эмоции».
- Модель безусловно встраивает в аудио неслышимый watermark (Perth); штатно не
  отключается. На стрим не влияет.
- **Установка чище, но не чистая.** См. следующий раздел — это главный риск.

## Риск №1: зависимости chatterbox-tts

Проверено по метаданным PyPI (`chatterbox-tts` 0.1.7): пакет **жёстко пинит**
`gradio==6.8.0`, `transformers==5.2.0`, `diffusers==0.29.0`, `librosa==0.11.0`
(тянет numba/llvmlite), а также `torch==2.6.0` / `torchaudio==2.6.0` для
Python < 3.14. Портативная сборка везёт Python 3.12.10
(`scripts/build_portable.py:35`), то есть попадает ровно под этот пин.

Отсюда два следствия, которые нельзя замалчивать:

- обещание «gradio уходит» **ложно**, если ставить пакет как есть: gradio со
  своим хвостом вернётся в gpu-runtime;
- откат `torch` 2.13 → 2.6 лишает поддержки **Blackwell (RTX 50xx, sm_120)** —
  на таких картах worker упадёт с «no kernel image», хотя сейчас работает.

**Решение (проверяется спайком, до всего остального):** ставить пакет с
`--no-deps` и держать в `RUNTIME_PACKAGES` явный список того, что worker
действительно импортирует (torch, torchaudio, transformers, diffusers, librosa,
s3tokenizer, resemble-perth, conformer, safetensors, omegaconf, numpy и
языковые токенизаторы). gradio worker не использует вовсе. Если с более свежим
torch движок не заводится — фиксируем torch 2.6.0 и **явно принимаем** потерю
Blackwell, записав это в README.

Пока спайк не дал ответа, `pins.py` писать нельзя — это будет гадание. Ровно на
этом обожглись с S1-mini (отсюда нынешние пины на `wandb` и `protobuf`).

## Архитектура

Каркас нынешней озвучки сохраняется — он верный:

- worker-подпроцесс держит модель в VRAM, общается по HTTP (`GET /health`,
  `POST /synth`);
- worker умирает по EOF stdin, поэтому сирота не удержит видеопамять;
- gpu-runtime ставится pip-ом в отдельное дерево, отпечаток пинов сторожит
  несовместимость;
- веса качаются с зеркала (GitHub Release) по sha256, докачка через Range;
- недоступность голоса — штатное состояние: реплики уходят текстом.

Не меняются: `director.py`, `tts/audio.py`, `tts/bootstrap.py` (кроме удаления
`.project-root`-костыля), `commentary/defaults.py` (промпт с маркерами остаётся
как есть).

### Где живёт разбор маркера

Ключевое решение, от которого зависит половина правок. Реплика от LLM приходит
как `(angry) Ну и куда ты поехал?`. Разбирается она **в двух местах, и это
намеренно**:

- `client.synth(text, voice)` — **сигнатура не меняется**. Внутри клиент сам
  зовёт `markers.parse`, снимает маркер, берёт по нему `(exaggeration,
  cfg_weight)` и шлёт в worker уже чистый текст плюс числа. Поэтому
  `POST /api/tts/preview`, который зовёт `tts.synth` напрямую
  (`server.py:347`), работает точно так же, как эфир — маркеры в превью можно
  слушать.
- `broadcast.publish` — зовёт `markers.parse` ещё раз (функция чистая, разбор
  дешёвый) ради двух вещей: чистый текст в оверлей зрителю и маркер для выбора
  голоса. В синтез при этом уходит **оригинальный текст с маркером** — как и
  сейчас; стиль с него снимет клиент.

Правило разбора: стиль задаёт **первый известный** маркер, где бы он ни стоял;
**все** известные маркеры вырезаются из текста. LLM просят ставить один маркер в
начале, но полагаться на это нельзя.

### Изменения по файлам

**`tts/pins.py`** — переписывается.
`RUNTIME_PACKAGES` — явный список (см. «Риск №1»), устанавливается с `--no-deps`.
`WEIGHTS` — манифест файлов Chatterbox Multilingual, `WEIGHTS_BASE_URL` — новый
релиз `models-chatterbox-v1`.

Состав весов (имена сверены с `from_local` в `src/chatterbox/mtl_tts.py`, точные
sha256 и размеры снимает спайк):

| Файл | Примечание |
|---|---|
| `t3_mtl23ls_v2.safetensors` | либо `v3` — выбор за спайком, см. ниже |
| `s3gen.pt` | для v3 уточнить, нужен ли `s3gen_v3.pt` |
| `ve.pt` | |
| `grapheme_mtl_merged_expanded_v1.json` | токенизатор; **не** `mtl_tokenizer.json` |

| `Cangjie5_TC.json` | `from_pretrained` его тянет; проверить, нужен ли `from_local` |

**`tts/bootstrap.py`** — удаляется создание `.project-root` (было нужно только
`pyrootutils` из fish-speech). Остальное без изменений.

**`paths.py`** — `MODEL_DIR` → `HOME / "models" / "chatterbox"`. Без этого веса
лягут в папку `models\s1-mini`, и ручная уборка снесёт их вместе со старьём.

**`tts/worker.py`** — переписываются `load_engine()` и `synth_wav()`:

```python
from chatterbox.mtl_tts import ChatterboxMultilingualTTS
model = ChatterboxMultilingualTTS.from_local(ARGS.model_dir, device="cuda")
wav = model.generate(text, language_id="ru", audio_prompt_path=ref_wav,
                     exaggeration=exaggeration, cfg_weight=cfg_weight)
```

Результат — torch-тензор; в WAV пакуется тем же `wave`-кодом, частота из
`model.sr`. Тело `POST /synth` получает новые поля `exaggeration` и
`cfg_weight`. `watch_parent_stdin`, `Handler`, `LOCK`, фазы — без изменений.
`language_id="ru"` зашит константой: приложение русскоязычное, в настройки не
выносится.

**`tts/client.py`** — класс `S1MiniTTS` → `ChatterboxTTS`. `synth(text, voice)`
сохраняет сигнатуру, но внутри разбирает маркер и кладёт в тело запроса
`exaggeration`/`cfg_weight` (см. «Где живёт разбор маркера»).

**`tts/markers.py`** — `EMOTION_MARKERS` сокращается до **четырёх** контрастных
маркеров: прослушивание спайка показало, что промежуточные градации (confident,
disdainful, laughing…) на слух неразличимы — широкий список размывал промпт без
выигрыша в звуке. Добавляются:

- `parse(text) -> tuple[str | None, str]` — маркер и очищенный текст;
- `MARKER_STYLE: dict[str, tuple[float, float]]` — маркер → (exaggeration,
  cfg_weight);
- `_LEGACY` — маркеры старого списка (`shouting`, `laughing`…): сохранённые в
  БД персоны могут их ставить, поэтому парсер их вырезает и приводит к
  ближайшему из четырёх (иначе движок зачитает маркер вслух).

`strip_markers` уходит: `broadcast.py` переходит на `parse`.

Таблица (значения разведены по углам после прослушивания):

| Маркер | exaggeration | cfg_weight |
|---|---|---|
| без маркера | 0.5 | 0.5 |
| angry (+shouting, disdainful) | 0.9 | 0.25 |
| excited (+laughing, chuckling, confident, surprised) | 0.75 | 0.35 |
| sad (+sighing) | 0.35 | 0.65 |
| whispering | 0.25 | 0.7 |

**`tts/voices.py`** — упрощается: транскрипт не нужен. `save_voice(name, wav)`
без `.txt`, `list_voices()` смотрит только на `*.wav`, `voice_paths()` отдаёт
один путь. `delete_voice` дополнительно сносит одноимённый `.txt`
(`missing_ok=True`) — иначе транскрипты уже сохранённых голосов останутся
сиротами. `pick_voice(settings, stim_type, priority, marker)` получает новый,
самый высокий уровень приоритета — референс, привязанный к маркеру.

**`tts/__init__.py`** — экспорты: `ChatterboxTTS` вместо `S1MiniTTS`, `parse`
вместо `strip_markers`.

**`main.py`** — импорт и конструктор `ChatterboxTTS` (строки 31, 178).

**`broadcast.py`** — импорт нового имени класса; `publish` зовёт
`markers.parse`, отдаёт зрителю чистый текст, передаёт маркер в `pick_voice`, а в
`synth` по-прежнему шлёт оригинальный текст.

**`config.py`** — новое поле `voice_by_marker: dict[str, str]` (маркер → имя
голоса).

**`server.py`** — из `POST /api/voices` уходит поле `transcript`; `SettingsIn`
принимает `voice_by_marker`.

**`web/src/shared/api.ts`** — `voice_by_marker` в `SettingsDto`, `uploadVoice`
без `transcript`.

**`web/src/panel/Panel.tsx`** — из формы загрузки голоса убирается поле
транскрипта (подсказка про длину референса — по результатам спайка); добавляется
блок «голос по маркеру» той же формы, что существующие `voice_by_priority` и
`voice_overrides`.

## Эмоции: как маркер становится звуком

1. `markers.parse()` → `("angry", "Ну и куда ты поехал?")`. Зрителю в оверлей —
   чистый текст.
2. Маркер даёт пару чисел из `MARKER_STYLE`. Нет маркера — `(0.5, 0.5)`.
3. Голос выбирается по цепочке приоритетов:
   `voice_by_marker[marker]` → `voice_overrides[stimulus.type]` →
   `voice_by_priority[priority]` → `default_voice`.

Смысл: **работает сразу, углубляется по желанию.** Ничего не записав,
пользователь получает разную подачу за счёт `exaggeration`. Записав `angry.wav`
и привязав его к маркеру в панели, получает и другой тембр. Имя голоса без файла
игнорируется на своём уровне — синтез не падает.

## Ударения (RUAccent)

Chatterbox русских ударений не знает и в омографах ошибается («замок»,
«дорога»). A/B на живой машине показал: модель **уважает знак ударения U+0301**
в тексте — произношение управляемо. Штатная интеграция `russian_text_stresser`
отвергнута (GPLv3, Python <3.12, llama-cpp в зависимостях). Решение —
**RUAccent** (Apache-2.0, PyPI, стандарт в русских TTS-пайплайнах):

- живёт в worker: `acc.process_all(text)` → `з+амок` → regex → `за́мок` →
  в `generate()`; вызов 0–172 мс на фразу (замер спайка), загрузка ~15 с один
  раз при старте worker;
- модель омографов `turbo`, `use_dictionary=True`; workdir —
  `MODEL_DIR / "ruaccent"`;
- данные (~465 МБ: словари + ONNX-модели) зеркалятся архивом
  `ruaccent-data.zip` в тот же релиз `models-chatterbox-v1`; bootstrap качает по
  sha256 и распаковывает — RUAccent не должен ходить на HF с машины
  пользователя;
- пакеты `ruaccent` + его зависимости (onnxruntime, razdel, python-crfsuite,
  sentencepiece, flatbuffers) идут в `RUNTIME_PACKAGES` обычной установкой —
  конфликтов с пинами нет (проверено спайком);
- ошибка аккцентизации не валит синтез: `try/except` → текст без ударений.
  Ударения — улучшение, а не точка отказа.

## Уборка старого

Приложение установлено только у автора, поэтому кода миграции не пишем — это
были бы строки, которые исполнятся один раз и никогда больше. Удаляется руками,
**после** успешного первого запуска на Chatterbox:

- `%LOCALAPPDATA%\StreamDirector\models\s1-mini\` (3.6 ГБ) — безопасно, потому
  что новые веса лежат в `models\chatterbox\`;
- `build\s1-mini\` (3.4 ГБ) и `build\cache\silero_v4_ru.pt` (реликт позапрошлой
  озвучки);
- `data\voices\*.txt` — транскрипты больше не читаются;
- GitHub-релиз `models-s1-mini-v1` — `gh release delete`.

`gpu-runtime\` сносится сам: `ensure_runtime` сравнивает отпечаток пинов, а они
меняются радикально.

## Обработка отказов

Не меняется — нынешняя логика верна: нет NVIDIA → `no_gpu` и текст вместо
голоса; веса не скачались или sha256 не сошёлся → `BootstrapError` с человеческим
текстом и кнопкой «повторить»; синтез упал → HTTP 500, worker жив; worker умер →
до трёх перезапусков с бэкоффом.

## Проверка

Тестов в проекте нет и мы их не заводим. Проверяем скиллом `verify`:

- приложение поднимается, бейдж «голос» зеленеет;
- `POST /api/tts/preview` даёт внятную русскую речь на встроенном голосе;
- то же с загруженным референсом — узнаётся тембр;
- маркеры слышны: `(whispering)` и `(shouting)` на одном тексте через тот же
  preview звучат по-разному (работает, потому что разбор живёт в `client.synth`);
- `GET /health` показывает VRAM; время синтеза фразы укладывается в
  `tts_max_age_s = 20`;
- реплика в эфире: зритель видит текст без маркера, слышит голос.
