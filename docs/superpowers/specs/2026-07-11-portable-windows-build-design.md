# Portable-сборка для Windows (доставка без Python/Node)

Дата: 2026-07-11

## Задача

Пользователь скачивает один zip со страницы GitHub Releases, распаковывает,
запускает `StreamDirector.exe` — сервер стартует, панель сама открывается
в браузере. Ставить Python, Node.js и зависимости не нужно; голос (Silero)
работает офлайн из коробки.

Решения, принятые с пользователем:
- канал доставки — публичные GitHub Releases;
- всё в комплекте, включая голос (torch + модель Silero), один артефакт
  ~600–900 МБ;
- формат — portable zip без инсталлятора и админ-прав;
- обновления — уведомление в панели со ссылкой, без автообновления;
- платформа — только Windows x64 (Mac/Linux — из исходников по README).

Вне объёма: подпись кода (SmartScreen будет предупреждать — репутация
наберётся со временем), автообновление, сборки под macOS/Linux, GPU-torch.

## Выбор подхода

Embedded Python (выбран) vs PyInstaller onedir vs Nuitka:

- **Embedded Python** — официальный embeddable-дистрибутив CPython + зависимости,
  поставленные pip'ом в CI + исходники проекта как есть. Torch работает в
  обычном окружении (ноль проблем с DLL и динамическими импортами), сборка
  прозрачна и отлаживаема, антивирусы не ругаются на официальный python.exe.
- PyInstaller отвергнут: torch под ним — хронический источник боли (hidden
  imports, DLL, регрессии хуков), а его bootloader чаще триггерит антивирусы —
  критично для публичной раздачи.
- Nuitka отвергнут: десятки минут компиляции, хрупкость с torch, а его плюсы
  (скорость, защита исходников) здесь не нужны.

## Структура дистрибутива

Артефакт релиза: `StreamDirector-vX.Y.Z-win64.zip`, распаковывается в:

```
StreamDirector/
├─ StreamDirector.exe        # лаунчер — маленький C-стаб (~20 КБ)
├─ python/                   # официальный embeddable CPython 3.12
│  ├─ python.exe, *.dll
│  ├─ python312._pth         # правится при сборке: включён import site
│  └─ Lib/site-packages/     # fastapi, uvicorn, torch (CPU) и пр.
├─ app/
│  ├─ src/stream_director/   # исходники проекта как есть
│  └─ web/dist/              # собранный фронтенд
├─ models/
│  └─ silero_v4_ru.pt        # модель Silero (скачана в CI, пиннед URL+SHA256)
└─ data/                     # создаётся при первом запуске: settings, БД, логи
```

Ключевая идея: сам проект **не** пакуется и не ставится pip'ом — он едет
папкой исходников, а в embedded Python ставятся только сторонние зависимости.
Окружение в рантайме идентично dev-режиму: `web/dist` находится по
существующему относительному пути (`server.py`, монтирование `web/dist`),
замораживать/распаковывать нечего.

Torch ставится CPU-only (`--index-url https://download.pytorch.org/whl/cpu`) —
Silero отлично работает на CPU, а CUDA-вариант добавил бы ~2 ГБ.

## Лаунчер `StreamDirector.exe`

C-стаб (`scripts/launcher.c`), компилируется MSVC прямо в CI. Делает ровно:

1. ставит CWD в директорию своего exe — чинит относительный `data/`
   из `main.py` (`DATA_DIR = Path("data")`);
2. выставляет `PYTHONPATH=app\src` и `STREAM_DIRECTOR_OPEN_PANEL=1`;
3. запускает `python\python.exe -m stream_director`, консоль с логами
   остаётся открытой;
4. при ненулевом коде выхода ждёт нажатия клавиши, чтобы пользователь успел
   прочитать ошибку (например, «порт занят»).

Браузер лаунчер **не** открывает — это делает Python после фактического
старта сервера, чтобы не поймать «connection refused» на медленном первом
запуске.

## Изменения в коде приложения

Все изменения обратно совместимы с dev-запуском `python -m stream_director`.

1. **Локальная модель Silero** (`tts.py`). Сначала ищем `models/silero_v4_ru.pt`
   относительно CWD; если файл есть — грузим через
   `torch.package.PackageImporter` без сети. Если нет (dev-режим) — текущий
   путь через `torch.hub.load` остаётся фолбэком.
2. **Автооткрытие панели** (`main.py`). После старта uvicorn, если выставлен
   `STREAM_DIRECTOR_OPEN_PANEL=1`, открываем
   `http://127.0.0.1:<port>/panel` стандартным модулем `webbrowser`.
   Dev-запуски без переменной браузер не трогают.
3. **Версия приложения**. `__version__` в `stream_director/__init__.py` —
   единственный источник правды; отдаётся в `/api/status`; CI проверяет
   совпадение с тегом релиза (и с версией в `pyproject.toml`).
4. **Проверка обновлений**. На старте один фоновый запрос к
   `https://api.github.com/repos/Lakai4eg/wot-ai-commentator/releases/latest`
   (httpx, таймаут ~5 с, любая ошибка молча игнорируется). Если версия
   новее текущей — `/api/status` включает
   `update_available: {version, url}`, панель показывает ненавязчивый баннер
   «Доступна версия X.Y.Z» со ссылкой на Releases. Без автоскачивания.

## Скрипт сборки — `scripts/build_portable.py`

Один скрипт, работает и локально, и в CI (Windows):

1. скачать embeddable CPython 3.12 (пиннед-версия + SHA256), распаковать
   в `python/`;
2. включить `import site` в `python312._pth`; зависимости поставить питоном
   раннера: `pip install --target python/Lib/site-packages .` + CPU-torch
   с индекса PyTorch;
3. `npm ci && npm run build` в `web/`; скопировать `src/` и `web/dist/`
   в `app/`;
4. скачать модель Silero (пиннед URL + SHA256) в `models/`;
5. скомпилировать `scripts/launcher.c` MSVC'ом в `StreamDirector.exe`;
6. собрать дерево и запаковать в zip.

## CI — GitHub Actions

Workflow на push тега `v*`:

1. pytest;
2. проверка: тег == `__version__` == версия в `pyproject.toml`, иначе fail;
3. `build_portable.py` → zip;
4. **смоук-тест**: распаковать, запустить, дождаться `/api/status` → 200
   с правильной версией и `tts_status: ready` (таймаут щедрый, ~5 мин) —
   это проверяет, что torch и модель реально работают в portable-окружении;
5. создать GitHub Release с приложенным zip и авто-сгенерированными нотами.

## Тестирование

- Юнит-тесты: сравнение версий в проверке обновлений; выбор пути модели
  (локальная vs hub) в `tts.py`.
- Интеграция: смоук-тест в CI (см. выше).
- Перед первым публичным релизом — один ручной прогон на машине без
  Python/Node.
