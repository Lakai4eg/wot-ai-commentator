# Озвучка на OpenAudio S1-mini (GPU): дизайн

Дата: 2026-07-12. Статус: одобрен пользователем (brainstorming, секции 1–6).

## Цель

Заменить Silero на OpenAudio S1-mini (fish-speech, 0.5B) — эмоциональный русский
голос на GPU. Проект некоммерческий, лицензия модели CC-BY-NC-SA 4.0 подходит.

## Зафиксированные решения

| Вопрос | Решение |
|---|---|
| Сосуществование с Silero | Silero удаляется полностью; голос требует NVIDIA GPU, без GPU приложение работает текстом |
| Голоса | `default` (тембр модели, без референса) + пользовательские референсы WAV+транскрипт через панель |
| Эмоции | LLM сам расставляет маркеры `(angry)`, `(laughing)` и т.п.; в оверлей текст идёт очищенным |
| Доставка стека (~5 ГБ) | Докачка при первом старте с лоадером; zip-дистрибутив остаётся маленьким |
| Архитектура | Тонкий worker-подпроцесс с отдельным site-packages (`gpu-runtime/`) |
| Реализация | План исполняется через workflows (запрос пользователя) |

## Архитектура

### Раскладка на диске (относительно CWD, его ставит лаунчер)

```
gpu-runtime/            site-packages воркера: CUDA-torch, fish-speech, зависимости
models/s1-mini/         веса модели (LLM + аудиокодек), SHA256-проверка
data/voices/            референсы: <имя>.wav + <имя>.txt (транскрипт)
```

### Bootstrap (первый старт, фоном; сервер и текст работают сразу)

1. **Проверка GPU**: `nvidia-smi` доступен и видит устройство. Нет → `state: no_gpu`,
   голос отключён, панель объясняет причину, кнопка «проверить снова».
2. **gpu-runtime**: если каталог неполный — `python -m pip install --target gpu-runtime …`
   (CUDA-индекс PyTorch + fish-speech, версии запинены в `bootstrap.py`).
   Прогресс грубый: парсим stdout pip, показываем текущий пакет.
3. **Веса**: скачивание с зеркала в GitHub Releases проекта (не HuggingFace — репозиторий
   fishaudio/openaudio-s1-mini гейтится, требует логина) с точным прогрессом
   (байты/всего) и SHA256. Состав: `model.pth` (1.74 ГБ), `codec.pth` (1.87 ГБ),
   `tokenizer.tiktoken`, `special_tokens.json`, `config.json` — каждый файл < 2 ГБ
   лимита GitHub-ассетов.
   Возобновляемо: `.tmp`-файлы, повторная проверка хешей при старте.

Повторные старты пропускают шаги 2–3 (всё на месте) и идут сразу к запуску worker.

### Worker-подпроцесс

- Запускается embedded-питоном приложения со путями `gpu-runtime/`
  (файл `worker.py` приложение не импортирует — только спавнит).
- Фазы: `starting` → `loading` (модель разворачивается в VRAM) → `ready`.
- Мини-HTTP на `127.0.0.1:<свободный порт>`:
  - `GET /health` → `{phase, vram_mb}`;
  - `POST /synth {text, voice}` → WAV bytes (нативная частота модели, 44.1 кГц).
    Один синтез за раз (lock).
- Референсы: клиент передаёт в `/synth` имя голоса; worker читает
  `data/voices/<имя>.wav|.txt` сам, `default` — синтез без референса.
- Жизненный цикл: parent держит stdin worker-а; EOF (родитель умер любым способом)
  → worker завершает себя. Штатное закрытие: `main.run()` в `finally` терминирует
  worker → VRAM гарантированно свободна (умирает процесс, а не тензор).

### Статусы для лоадеров

`ctx.statuses["tts_state"]` (новое поле; существующие `tts: bool` для бейджа и
строковый `tts_status` сохраняются, `tts_status` дублирует `state`):

```
{ state: checking | no_gpu | downloading_runtime | downloading_model
        | starting | loading | ready | error,
  progress: {done_mb, total_mb} | {step: "имя пакета"},
  error: строка | null }
```

Панель поллит `/api/status` раз в 2 с (уже есть). Пока state ≠ ready — баннер-лоадер
вверху панели: шаги pip → прогресс-бар весов → спиннер «разворачиваю модель в память
видеокарты». Бейдж «голос» в шапке остаётся. Оверлей (OBS, эфир) лоадер не показывает.

## Компоненты

- `src/stream_director/tts/` (пакет вместо файла `tts.py`):
  - `bootstrap.py` — GPU-проверка, докачки, пины версий, URL+SHA256 весов, статусы;
  - `client.py` — `S1MiniTTS`: спавн/рестарт/остановка worker, `synth(text, voice)
    → bytes|None` (sync, зовётся через `asyncio.to_thread` как раньше), `available`;
  - `worker.py` — точка входа подпроцесса;
  - `voices.py` — список/загрузка/удаление референсов в `data/voices/`;
  - `markers.py` — список поддерживаемых эмо-маркеров, вырезание их из текста.
- `broadcast.py`: в синтез — текст с маркерами, в оверлей/панель — очищенный.
- `config.py`: `default_voice: "default"`; `voice_by_priority`/`voice_overrides`
  сохраняют семантику (значения — имена референсов). Несуществующее имя → `default`
  (деградация как в текущем `pick_voice`).
- `server.py`: `/api/voices` — `default` + референсы; новые `POST /api/voices`
  (multipart: name, wav, text) и `DELETE /api/voices/{name}`; `/api/tts/preview`
  без изменений. `/api/status` отдаёт объект в новом поле `tts_state`
  (bool-поле `tts` остаётся для бейджа).
- `main.py`: вместо `load_tts()` — bootstrap-цепочка в фоне; `finally` — остановка
  worker.
- Панель (`Panel.tsx`, `api.ts`): баннер-лоадер, секция «Голос» + загрузка
  референса (имя, WAV-файл, транскрипт), удаление голоса.
- Удаляются: `SileroTTS`, константа `VOICES`, silero-модель из сборки.

## Эмоции

- В заводской `response_format` (`commentary/defaults.py`) — инструкция LLM:
  можно ставить маркеры из фиксированного списка (~8–10 штук: angry, excited,
  laughing, sad, whispering, shouting и др. — финальный отбор по прослушиванию
  на этапе реализации; полный список из 48 в промпт не идёт).
- `markers.py` вырезает только известные маркеры; прочие скобки не трогает.
- Персоны пользователей не редактируются — инструкция живёт в response_format,
  общем для всех персон.

## Ошибки и граничные случаи

- **Нет NVIDIA/CUDA** → `no_gpu`, текст работает, панель: «Для голоса нужна
  видеокарта NVIDIA (6+ ГБ)».
- **Обрыв докачки** → продолжение при следующем старте (SHA256 решает, что качать);
  кнопка «повторить» в баннере при `error`.
- **Worker упал** (CUDA OOM — например VRAM занята игрой; краш драйвера) → `error`
  с причиной, авторестарт с бэкоффом (паттерн `supervised()`); после 3 неудач
  подряд — ждём ручного «повторить». Приложение и текст живут.
- **Опоздавшая озвучка** — существующий `tts_max_age_s` не меняется.
- **Зомби-worker** — исключён: смерть родителя = EOF stdin = самозавершение;
  порт выбирается свободный на каждый старт.

## Сборка, лицензия, релиз

- `build_portable.py`: убрать silero-модель и установку `[ml]`; добавить `pip`
  в бандл (нужен бутстраперу). Zip ~60–100 МБ вместо 247 МБ.
- `pyproject.toml`: extra `ml` удаляется; зависимости worker-а живут пинами
  в `bootstrap.py`, не в метаданных проекта.
- Релиз: веса S1-mini — ассеты GitHub-релиза (каждый файл < 2 ГБ);
  README — атрибуция: OpenAudio S1-mini © Fish Audio, CC-BY-NC-SA 4.0,
  использование некоммерческое.

## Верификация

- Тестов в проекте нет и не создаются (правило пользователя).
- Skill `verify`: E2E с фейковыми источниками — реплика доходит до overlay-WS;
  статусная цепочка bootstrap видна в `/api/status`.
- Ручной прогон на машине с GPU: холодный старт (оба лоадера), синтез с эмоцией,
  превью голоса, закрытие приложения → `nvidia-smi` подтверждает освобождение VRAM.

## Риски и первый шаг реализации

Главный риск — совместимость fish-speech с embedded Python 3.12 на Windows
(пины transformers, отсутствие компилятора у пользователя, поведение
`--target`-установки). Поэтому **шаг 0 плана — spike**: на dev-машине поставить
стек в изолированный target-каталог, синтезировать русский WAV с эмо-маркером,
зафиксировать пины и источник пакетов (PyPI или git-тег) в `bootstrap.py`.
Если embedded-питон не подойдёт — worker получает собственный полный CPython
(скачивается бутстрапером так же, как всё остальное); архитектура не меняется.

Ожидания по ресурсам пользователя: ~6.5 ГБ докачки на первом старте (рантайм ~3 ГБ
+ веса ~3.6 ГБ), ~12 ГБ на диске, ~3–4 ГБ VRAM у worker-а, синтез короткой фразы
на RTX 3060 — единицы секунд.
