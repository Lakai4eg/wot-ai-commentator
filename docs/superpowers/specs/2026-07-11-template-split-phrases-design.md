# Разделение шаблонных событий LoL и наполнение фраз до 20

Дата: 2026-07-11. Статус: утверждено пользователем.

## Цель

Шаблонные реплики (`src/stream_director/games/lol/templates/*.txt`) озвучиваются на
стриме при игровых событиях. Нужно:

1. Разделить обобщённые события на конкретные файлы:
   - объекты → дракон / барон / герольд × забрали / отдали + крады;
   - башни → лично стример / взяла команда / потеряли свою.
2. Дополнить каждый файл до **20 фраз** (строка = фраза, `#` — комментарий).

## Тон фраз

Сарказм и подколки, **лёгкая токсичность допустима**. Мемы в духе: «пресс F»,
«клипаем», «чат уже печатает», смайт-дифф, «0/10 пауэрспайк», «гап», «инт»,
«мид дифф». Твич-мемы — русифицированные («кекв», а не «KEKW»): фразы читает TTS,
латинские эмоуты он произносит буквально. Лексика LoL умеренно (ферма, ганк,
вард, пуш, фид, скейлинг). Без плейсхолдеров `{…}` — шаблон уходит в TTS как есть.

## Структура файлов (26 файлов × 20 фраз)

### Объекты

`objective_ours.txt` / `objective_theirs.txt` **удаляются**, их фразы переезжают
в новые файлы (где уместно) и дополняются:

| Файл | Событие |
|------|---------|
| `objective_dragon_ours` / `objective_dragon_theirs` | дракон взяли / отдали |
| `objective_baron_ours` / `objective_baron_theirs` | барон взяли / отдали |
| `objective_herald_ours` / `objective_herald_theirs` | герольд взяли / отдали |
| `objective_stolen_ours` / `objective_stolen_theirs` | крад наш / крад у нас |
| `objective` | фолбэк: сторона неизвестна (`side == "unknown"`) |

Крад (`stolen`) имеет приоритет над видом объекта: украденный барон →
`objective_stolen_ours`, не `objective_baron_ours`. Файлы крадов общие для всех
видов объектов (фразы не называют конкретный объект).

### Башни (выбор пользователя: «стример отдельно + команды»)

| Файл | Событие |
|------|---------|
| `turret` | башню добил лично стример (текущее поведение и фразы) |
| `turret_ours` | вражескую башню снесла команда (союзник/миньоны) |
| `turret_theirs` | наша башня потеряна |

### Остальные 15 файлов

Не переименовываются, дополняются до 20 фраз каждый:
`ace_ours`, `ace_theirs`, `ally_carrying_lead`, `ally_carrying_multikill`,
`ally_feeding`, `assist`, `battle_start`, `death`, `first_blood`, `frag`,
`inhib`, `multikill`, `team_gap_behind`, `team_gap_spectator`, а также
`objective` (фолбэк).

## Правки кода

### `mapper.py`

- В payload события `objective` добавляется машинный ключ `kind_key`
  (`"dragon" | "baron" | "herald"`). Русская строка `kind`
  («дракон (Earth)») остаётся для LLM-описаний; ветвиться по ней хрупко.
- `TurretKilled`: событие эмитится для **всех** снесённых башен, не только
  стримерских. Сторона определяется по имени снесённой башни
  (`Turret_T1_…` = ORDER, `Turret_T2_…` = CHAOS) против команды стримера:
  вражеская башня пала → `side="ours"`, наша → `side="theirs"`. Это надёжнее,
  чем `_side_of(KillerName)`: башни часто добивают миньоны, которых
  `_side_of` не сопоставит (вернёт `unknown`). Если добил лично стример —
  прежнее событие без стороны (payload `{}` → файл `turret`).
  Если команда башни из имени не распознана и добил не стример — событие
  с `side="unknown"` не эмитится вовсе (не врём про сторону — прецедент
  ложных комментариев про дракона уже был).
- `InhibKilled` не трогаем (пользователь не просил).

### `flavor.py`

- `variant_key` для `objective`: сначала `stolen` →
  `objective_stolen_{side}`, затем `objective_{kind_key}_{side}`; при
  `side == "unknown"` — прежний фолбэк `objective`.
- `variant_key` для `turret`: `turret_{side}` при наличии стороны в payload,
  иначе `turret`.
- `_EVENT_DESCRIPTIONS`: добавить `turret_ours` («команда стримера снесла
  башню противника — лично стример ни при чём») и `turret_theirs`
  («противник снёс башню команды стримера»). Описание `objective` уже
  покрывает вид/сторону/крад через `side_ru`/`kind`/`stolen_note`.

### Тесты

Обновить `test_lol_mapper.py` (TurretKilled по сторонам, kind_key),
`test_lol_flavor.py` (новые ветвления variant_key, описания turret_*).
`test_template_pool.py` не зависит от конкретных файлов — не трогаем.

## Генерация фраз — Workflow

Fan-out: по агенту на файл (26), каждому — контекст события, существующие
фразы файла (сохранить и не дублировать), требования тона. Затем сводный
проход: дедуп слишком похожих фраз между файлами и проверка счёта = 20.

## Критерии готовности

- 26 файлов, в каждом ровно 20 непустых строк-фраз (комментарии не в счёт).
- `objective_ours.txt` / `objective_theirs.txt` отсутствуют, их фразы
  сохранены в новых файлах.
- Существующие фразы всех файлов сохранены дословно.
- Тесты проходят (`pytest`).
- Без плейсхолдеров и латинских эмоутов в фразах.
