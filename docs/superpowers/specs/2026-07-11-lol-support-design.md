# Поддержка League of Legends — дизайн

Дата: 2026-07-11. Статус: утверждён (обсуждение в сессии brainstorming).

## 1. Цель

Добавить вторую игру — League of Legends — в AI-режиссёра стрима по той же
схеме, что и «Мир танков»: локальный API игры → события → стимулы → директор →
LLM-реплика → оверлей + голос. Обе игры работают одновременно с автодетектом:
какая игра живая, ту и комментируем; переключение без единого клика.

Решения, принятые на брейншторме:

- **Автодетект** активной игры (оба адаптера всегда запущены и переподключаются
  фоном), без ручного переключателя.
- **Только внутриигровые данные** LoL v1: Live Client Data API (порт 2999).
  LCU API (чемпселект, лобби, послематчевый экран) — вне скоупа.
- **Один персонаж, пер-игровой колорит**: общее ядро персоны (ехидный, но
  добрый закадровый ИИ-режиссёр, правила юмора), сленг и мишени — от модуля игры.
- Имя пакета `wot_ai_commentator` не переименовываем (чистый churn; косметика —
  отдельно и потом).

## 2. Архитектура: игровые модули

Ядро (директор, чат, LLM-бэкенды, TTS, сервер/оверлей) остаётся
игро-независимым. Всё игро-специфичное собирается в `games/<id>/`:

```
src/wot_ai_commentator/
  games/
    wot/            # переезд из wotstat/ + вынос WoT-специфики
      client.py     #   DataProviderClient (как есть)
      mapper.py     #   EventMapper (как есть)
      memory.py     #   сегодняшние SessionMemory/BattleMemory
      flavor.py     #   описания событий, сленг-блок, шаблоны-фолбэки
      module.py     #   сборка объекта-модуля WoT
    lol/
      client.py     #   поллер Live Client Data API (httpx)
      mapper.py     #   снапшоты/события → Stimulus
      memory.py     #   боевая/сессионная память LoL
      flavor.py     #   описания событий, сленг, фолбэки LoL
      module.py     #   сборка объекта-модуля LoL
  # ядро без изменений структуры: director.py, events.py, chat/, commentary/,
  # tts.py, server.py, session_memory.py удаляется (переезд в games/wot/memory.py)
```

### 2.1 Контракт модуля игры

Duck-typing, без ABC — в духе того, как `EventMapper` уже принимает клиента.
`games/<id>/module.py` собирает один объект с атрибутами:

| Атрибут | Тип / сигнатура | Назначение |
|---|---|---|
| `id` | `str` («wot» / «lol») | ключ маршрутизации |
| `display_name` | `str` | панель/статус |
| `source` | `run()`, `stop()`, `status`, диагностика | клиент-транспорт |
| `memory` | `register(stim) -> list[str]`, `battle_lines()`, `session_lines()`, `summary_lines()` | память игры |
| `describe_event(stim)` | `-> str` | описание события для промпта |
| `flavor_lines()` | `-> str` | сленг/мишени — блок в промпт |
| `fallback_line(stim)` | `-> str \| None` | шаблон при недоступной LLM |
| `always_speak_types` | `frozenset[str]` | события в обход кулдауна |

`always_speak_types`: WoT — `{"death"}`; LoL — `{"death", "multikill"}`.

### 2.2 Stimulus.game

`Stimulus` получает поле `game: str`. Маппер штампует свой `id` — директор
узнаёт, чью память/колорит использовать, без глобального переключателя в
горячем пути. Чат-заказы поле не заполняют — их игра определяется трекером в
момент обработки.

### 2.3 ActiveGameTracker

Небольшой класс в ядре. Оба source всегда запущены и вечно переподключаются
(как сегодняшний WoT-клиент в статусе «waiting»).

- Игра становится **активной**, когда её source оживает: WoT — получен `init`
  по WebSocket; LoL — порт 2999 ответил (он существует только во время матча).
- Активная игра **остаётся активной, пока не оживёт другая** — между матчами
  LoL порт умирает, но заказы из чата продолжают относиться к LoL.
- Если ни одна игра ещё не была живой — по умолчанию `wot`.

Трекер нужен только там, где стимул не привязан к игре: чат-заказы
(`!roast`/`!hype`/`!stats` — память и колорит активной игры), бейдж «активная
игра» в панели. Игровые события в трекер не ходят — у них есть `stimulus.game`.

Сессионная статистика игр **раздельная**: у каждого модуля своя память,
`!stats` озвучивает сессию только активной игры.

## 3. LoL: источник событий

### 3.1 Клиент (`games/lol/client.py`)

Asyncio-поллер Riot Live Client Data API:

- `GET https://127.0.0.1:2999/liveclientdata/allgamedata` через
  `httpx.AsyncClient(verify=False)` — сертификат самоподписанный, порт
  существует только во время матча.
- Интервал: **1 с в игре, ~3 с в ожидании** (connection refused → статус
  `"waiting"`, в духе бэкоффа WoT-клиента).
- Один эндпоинт даёт всё: `activePlayer` (riot id, уровень, ХП, голда),
  `allPlayers` (чемпион, команда, scores: kills/deaths/assists/CS, `isDead`),
  `events` (append-only список с `EventID`), `gameData` (карта, режим,
  `gameTime`).

Клиент — только транспорт: держит последний снапшот, отдаёт каждый свежий
payload в коллбек, наружу — `run()/stop()/status`. Публичный метод
`handle_payload(data)` — для тестов без HTTP (зеркало `handle_message`
WoT-клиента).

### 3.2 Маппер (`games/lol/mapper.py`)

Потребляет снапшоты, эмитит стимулы с `game="lol"`. Два механизма:

1. **Журнал событий**: обрабатываем только события с `EventID` больше
   последнего виденного; курсор сбрасывается на новой игре (событие
   `GameStart` или `gameTime` пошёл назад). Стример определяется сравнением
   riot id из `activePlayer` с `allPlayers` (фолбэк — легаси `summonerName`).
2. **Дельты снапшота**: то, чего нет в журнале — низкий ХП (тихое, раз за
   жизнь), смерть/респаун по `isDead` как страховка.

Маппинг событий (404 на экране загрузки и частичные payload'ы молча
пропускаются):

| Событие Live Client | Стимул | Приоритет |
|---|---|---|
| `GameStart` | `battle_start` (карта/режим/чемпион, тихое) | — |
| `ChampionKill`, киллер — я | `frag` (чемпион жертвы) | HIGH |
| `ChampionKill`, жертва — я | `death` (чемпион убийцы) | HIGH, always-speak |
| я в `Assisters` | `assist` | LOW |
| `Multikill` (мой) | `multikill` (дабл…пента) | HIGH, пента — CRITICAL, always-speak |
| `FirstBlood` (я с любой стороны) | `first_blood` (кто) | HIGH |
| `DragonKill`/`BaronKill`/`HeraldKill` | `objective` (тип, ours/theirs, флаг stolen) | NORMAL, stolen — HIGH |
| `TurretKilled`/`InhibKilled` мной | `turret` / `inhib` | NORMAL |
| `Ace` | `ace` (ours/theirs) | HIGH |
| `GameEnd` | `battle_result` (win/loss, тихое) | — |

Сознательно **не в v1**: комментарии про голду/CS, ап уровней, покупки
предметов — шум; от всплесков того, что эмитим, защищают уже существующие
дебаунс и кулдаун директора.

### 3.3 Память (`games/lol/memory.py`)

Интерфейс тот же, что у WoT. Боевой масштаб: чемпион, K/D/A, мультикиллы,
взятые объекты, чемпионы-убийцы. Сессионный: игр/побед, суммарный KDA, пенты,
«главный обидчик» по чемпиону (зеркало шутки WoT про убийц), факты вида «это
уже 3-я смерть от Дариуса за сессию».

## 4. Изменения ядра

### 4.1 Директор

Получает словарь модулей `{id: module}` + трекер вместо одной памяти. В
`process_once` резолвит модуль из `stimulus.game` (чат-заказ — активная игра
из трекера), дальше использует `memory`, `describe_event`, `fallback_line`,
`always_speak_types` этого модуля. Очередь, кулдаун, дебаунс, мьют — без
изменений (уже игро-независимы).

### 4.2 Промпты (`commentary/prompts.py`)

Общее ядро персоны (характер, правила юмора, правила ответа) остаётся здесь —
один источник истины. Сборка:

```
ядро персоны + module.flavor_lines() + контекст боя/сессии + описание события
```

WoT-колорит переезжает в `games/wot/flavor.py` дословно. LoL-колорит — свой
блок сленга и мишеней (тиммейты и джунглер — благодарные мишени, ферма, ганки,
варды); шутки общие, без хардкода мемов про конкретных чемпионов.
`_EVENT_DESCRIPTIONS` и `_TEMPLATES` переезжают в соответствующие `flavor.py`.

### 4.3 Настройки

Одно новое поле: `lol_url: str = "https://127.0.0.1:2999"`. Флагов
включения/выключения игр нет — автодетект означает, что оба адаптера просто
работают (YAGNI).

### 4.4 Сервер / панель

`/api/status`: блок `lol` в форме сегодняшнего `wotstat` (status,
events_found, last_events) + поле `active_game`. Панель: бейдж `wotstat`
становится двумя бейджами источников (WoT / LoL), активный подсвечен. Оверлей
без изменений.

### 4.5 Обработка ошибок

- 404 экрана загрузки и частичные payload'ы — молча пропускаем.
- Исключение маппера не убивает полл-луп (паттерн «шоу продолжается», как в
  `_emit`).
- Курсор `EventID` сбрасывается на новой игре — второй матч не проигрывает
  события заново и не пропускает их.

## 5. Тестирование

В стиле существующих тестов (pytest, фейковые клиенты, без сети):

- переезд/переименование WoT-тестов (`test_wotstat_*` → games/wot);
- LoL-клиент: `handle_payload` на заготовленном JSON, статусы waiting/connected;
- LoL-маппер: сценарии kill/death/multikill/objective, сброс курсора между
  матчами, определение стримера по riot id и фолбэк на summonerName;
- LoL-память: факты, боевые/сессионные строки;
- промпты: композиция «ядро + колорит», переключение колорита по модулю;
- директор: маршрутизация по `stimulus.game`, чат-заказ уходит в активную игру;
- трекер: правила активации (жив → активен; остаётся активным до оживания
  другой игры; дефолт wot).

## 6. Вне скоупа v1

- LCU API (чемпселект, лобби, послематчевый экран).
- Комментарии про голду/CS/уровни/предметы.
- Ручной переключатель активной игры в панели.
- Переименование пакета/репозитория.
