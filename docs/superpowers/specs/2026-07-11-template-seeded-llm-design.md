# Шаблоны как затравка для LLM (LoL) — дизайн

Дата: 2026-07-11. Статус: одобрен пользователем по секциям.

## Цель

Заготовленные реплики LoL (`_TEMPLATES` в `games/lol/flavor.py`) перестают быть
только аварийным фолбэком: они становятся материалом для реплик. Каждый шаблон
используется **один раз за сессию стрима** (сброс — перезапуск приложения),
после исчерпания запаса на событие LLM шутит без основы, как сейчас.

Охват: только LoL. WoT не трогаем (его `fallback_line` остаётся как есть),
но механизм кладём в ядро — подключение WoT позже не потребует рефакторинга.

## Режимы (`Settings.template_mode`)

Строка `"seed" | "verbatim" | "off"`, дефолт `"seed"`. Меняется на лету из
веб-панели (живой объект `Settings`, как `twitch_channel`).

- **`seed`** (дефолт): неиспользованный шаблон вставляется в промпт как
  «заготовка шутки» — LLM разворачивает её под контекст боя. Случайный
  «угол шутки» при этом в промпт не добавляется (затравка сама задаёт угол).
  Если LLM умерла — в эфир идёт сама затравка дословно (второй шаблон из
  пула не тратится). Пул на событие пуст → текущее свободное поведение.
- **`verbatim`** («шаблоны предпочтительно»): неиспользованный шаблон идёт в
  эфир дословно, LLM не вызывается. Пул пуст → свободная LLM-генерация.
- **`off`**: генерация как сейчас; шаблоны — только фолбэк при мёртвой LLM.

Пул общий для всех употреблений: шаблон, прозвучавший дословно (verbatim или
фолбэк) либо как затравка, помечается использованным и больше не всплывает.

Заказы из чата (`chat_order`/`dir`) шаблоны не затрагивают ни в одном режиме.

## Хранение шаблонов

Новая папка `src/stream_director/games/lol/templates/` — по `.txt`-файлу на
вариантный ключ события (`frag.txt`, `death.txt`, `objective_ours.txt`, …,
`team_gap_behind.txt`). Формат: одна строка = один шаблон; пустые строки и
строки, начинающиеся с `#`, игнорируются; кодировка `utf-8-sig` (BOM от
Блокнота переваривается). Содержимое `_TEMPLATES` переезжает в файлы,
словарь из `flavor.py` удаляется. Реплики можно дописывать без правки кода.

Папка включается в пакет как package data (`pyproject.toml`) — иначе она не
попадёт в portable-сборку.

## Компоненты

### `games/template_pool.py` — класс `TemplatePool`

- Конструктор: путь к папке шаблонов + функция вариантного ключа
  (`Callable[[Stimulus], str]`). Загружает файлы один раз; ключ = имя файла
  без расширения.
- `take(stimulus) -> str | None` — случайный неиспользованный шаблон по ключу;
  помечает использованным (на всю сессию); при пустом точном ключе откат на
  общий (`objective_ours` → `objective`); `None`, когда всё исчерпано.
- `exhausted_pick(stimulus) -> str | None` — выбор из полного списка с
  антиповтором последних трёх (текущая логика `_pick`) — аварийный фолбэк
  при мёртвой LLM после исчерпания пула.
- Состояние в экземпляре; экземпляр создаётся в `lol/module.py`.

### `games/base.py`

`GameModule` получает опциональное поле `template_pool: TemplatePool | None
= None`. LoL передаёт свой пул, WoT — `None`.

### `director.py`

Для игровых событий активного модуля с пулом, по `settings.template_mode`:

- `verbatim`: `pool.take()` → строка есть → в эфир дословно без LLM;
  строки нет (пул исчерпан) → LLM без затравки; LLM умерла →
  `exhausted_pick()`.
- `seed`: `seed = pool.take()`; есть → в `build_prompt(seed_line=seed)`;
  LLM вернула `None` → в эфир сам `seed`. Нет затравки → свободная
  генерация; LLM умерла → `exhausted_pick()`.
- `off`: как сейчас, но фолбэк мёртвой LLM: сначала `pool.take()`
  (расходует общий пул), затем `exhausted_pick()`.

Выбранная строка в любом режиме попадает в `_recent_replicas`; рейт-лимиты,
дебаунс и проверка протухания реплики не меняются.

### `commentary/prompts.py`

`build_prompt(..., seed_line: str | None = None)`. При наличии затравки
добавляется секция: «Заготовка шутки: „…“. Разверни её в свою реплику:
сохрани соль, адаптируй под текущий контекст боя, можешь перефразировать» —
и пропускается случайный «угол шутки».

### `lol/flavor.py`

`_TEMPLATES`, `_recent_picks`, `_pick`, `fallback_line` удаляются/переезжают
в `TemplatePool`. `_variant_key` остаётся и передаётся пулу. Контрактное
`fallback_line` LoL-модуля делегирует в пул (`take` → `exhausted_pick`).

### Настройки и панель

- `Settings.template_mode: str = "seed"`.
- `server.py`: поле в `SettingsIn`, валидация значения (400 при неизвестном,
  как `llm_provider`).
- `load_settings`: неизвестное значение в JSON → откат на `"seed"`.
- Панель (`web/src/panel/main.tsx`): переключатель на три положения
  (затравка / дословно / выкл), работает на лету.

## Обработка ошибок

- Папки нет / пуста → пустой пул, `take()` всегда `None`, warning в лог,
  поведение эквивалентно свободной генерации. Ничего не падает.
- Нечитаемый файл → пропуск с warning, остальные ключи работают.
- Старый `settings.json` без поля → дефолт `"seed"` автоматически.

## Тесты

- `tests/test_template_pool.py`: загрузка папки (`tmp_path`), игнор пустых
  строк/`#`-комментариев, BOM; `take()` без повторов за сессию и `None` при
  исчерпании; откат `objective_ours` → `objective`; `exhausted_pick()` не
  повторяет последние выданные.
- Директор: `verbatim` не зовёт бэкенд, пока пул жив; `seed` кладёт заготовку
  в промпт и убирает «угол шутки»; `None` от бэкенда в `seed` → в эфир сама
  затравка; `off` — прежнее поведение.
- `tests/test_lol_flavor.py`: проверки шаблонов переезжают на пул с файлами.
- Финально: полный прогон тестов; опционально E2E (skill `verify`).
